## 제품 한 줄
**혼자 사는 강원 시골 어르신에게 응급상황이 생기면, AI가 자동으로 탐지하고 → 반응이 없으면 119에 자동 신고 + 가까운 마을 이웃 3~4명을 즉시 호출해, 구급차가 도착하기 전까지 이웃이 초기 대처(프로토콜)를 하도록 돕는 마을 공동체 기반 응급 대응 앱.**

## 무슨 문제를 푸나 (Problem)
강원은 도시보다 **병원까지 물리적 거리가 멀어**, 응급상황 시 **탐지 → 신고 → 구급차 도착**에 시간이 너무 오래 걸린다. 특히 **혼자 사는 어르신**은 쓰러져도 **주변에 그 사실을 알 사람이 없어** 초기 대처가 늦고 고독사로 이어진다.
→ 우리는 **"발견 지연"과 "초기 대처 공백"** 두 지점을 공략한다. (거리 자체는 못 줄이지만, 골든타임 낭비를 줄인다.)

**근거(검증 완료 · 딥리서치):**
- 심정지(OHCA) 생존은 시간에 강하게 의존 — **이송 1분 지연당 생존 −6%**, 현장 초기대처(ROSC·CPR)가 생존을 결정.
- 한국 **농촌 OHCA 생존퇴원 3.4% vs 도시 8.7%** (2.5배 격차), 신고~도착 30분 초과 시 즉시사망 오즈 ↑.
- 강원 **30분 내 응급의료 접근 인구 79.5%**(전국 94%), **군지역 46%**, 18개 시군 중 **15곳 응급의료 취약지**.
- 강원 급성심정지 생존율 **7.2%(전국 최저)**, 일반인 CPR 시행률 도·군 **19.9%** vs 대도시 48.2% → "목격자 처치 공백".
- **상세 수치·출처(검증 완료):** [`../../screening/`](../../screening/README.md) (SDOH·산악수난·구급이송 병목 딥리서치 + 소스 50여 개). 응급처치 로직은 [`03-logic/02-first-aid-protocol.md`](../03-logic/02-first-aid-protocol.md).

## 핵심 가정 (Assumptions)
- **마을 공동체가 서로 돕는다.** 공동체 구성원이 모두 앱을 설치한다 (지자체 지원 하에 보급).
- 사용자에 **어르신이 많다** → 앱은 **직관적이고 쉬워야** 한다 (큰 글씨·고대비·최소 조작·음성).
- 현재는 홈캠 장비가 없어 **휴대폰 1대를 고정 위치에서 촬영**해 영상 입력으로 사용.

## 사용자 (두 입장 모두 설계)
| 역할 | 누구 | 앱에서 하는 일 |
|------|------|----------------|
| **환자 본인** | 혼자 사는 어르신 | (보호자 도움·동의로) 개인정보·거주지·병력 입력 / 응급 탐지 시 15초 알림 확인·해제 |
| **주변인(이웃)** | 근처 거주 마을 구성원 3~4인 | 호출 알림 수신 → 앱 열기 → 환자에게 찾아가 **프로토콜** 수행 |
| 보호자 | 가족 등 | 환자 대신 정보·병력 입력, 동의 관리 |
| (지자체) | 시군·보건소 | 앱 보급·공동체 등록 지원 (외부 주체) |

## 솔루션 — 한 앱에 두 축

### 1) 홈캠(폰 카메라) 영상인식 기반 응급 자동 탐지
- 고정된 휴대폰 카메라 영상에서 **쓰러짐(낙상) 등 응급상황을 AI가 자동 탐지.**
- 탐지되면 → 해당 어르신 앱에서 **약 15초간 큰 소리 알림음** 재생.
  - **터치로 해제** → 이상 없음 (오탐/일시적).
  - **무반응** → **119 자동 신고 + 아래 2) 실행.**
- ⚙️ 쓰러짐 탐지 모델은 **다른 팀원이 맥북 로컬로 구현 중** (현재 백엔드 서버 대신 로컬 처리).

### 2) 마을 공동체 기반 주변인 호출 + 프로토콜
- 응급 탐지 시 **가까운 거주지 이웃 3~4명 앱에 소리 알림** 전송.
- 알림 클릭 → 앱 열림 → **환자 주소 · 찾아가는 방법 · 조치사항(프로토콜)** 을 직관적으로 표시.
- **프로토콜 = O/X 퀴즈형 진단 + 사전 정의된 기본 조치 리스트.**
  - 환자(또는 보호자)가 입력한 **병력 기록 기반으로 LLM(Claude)이 조치 우선순위를 선정**해 주변인에게 제시.
- 도착한 주변인에게 앱이 **"음성으로 상황을 설명해 주세요"** 요청 → 짧은 음성 답변을 **Claude API로 처리**해 상태 파악.
- (추후 구현) 주변인의 프로토콜 수행 결과를 **출동 중 소방대원에게 실시간 전달.**

#### 프로토콜 로직 (초안)
1. **1단계는 항상 "호흡 여부"를 O/X로 질문.**
   - **O(호흡 있음)** → "음성으로 상황을 설명해 주세요" 요청 → 짧은 음성 답변을 Claude로 처리.
   - **X(호흡 없음)** → **CPR 실시 프로토콜**로 분기.
2. 이후 단계는 병력 기반 LLM 우선순위에 따라 사전 정의 리스트에서 제시. (세부는 [`03-logic/`](../03-logic/README.md)에서 정의)

## 핵심 플로우 (해피 패스)
```
[폰 카메라] 쓰러짐 탐지(로컬 AI)
   → 환자 앱: 15초 큰 알림음
      ├─ 터치 해제 → 종료(이상 없음)
      └─ 무반응
          → ① 119 자동 신고
          → ② 이웃 3~4명 앱 소리 알림
                → 이웃: 알림 클릭 → 앱 열림
                   → 환자 주소·접근법·프로토콜 표시(병력 기반 LLM 우선순위)
                   → 이웃 도착 → 앱: "음성으로 상황 설명" 요청 → Claude 처리
                   → 프로토콜 수행: [호흡 O/X]
                        ├─ O → 상황 음성 설명 처리
                        └─ X → CPR 프로토콜
                   → (추후) 결과를 출동 소방대에 실시간 전달
```

## 스코프 (MVP In / Out)
> **원칙: 완전한 앱보다 "심사위원이 눈으로 확인 가능한 시나리오 시연"이 최우선.**

**In (데모에 반드시):**
- 환자 앱: 정보·병력 입력 화면, 15초 응급 알림+해제.
- 주변인 앱: 호출 알림 수신 → 앱 열림 → 주소·접근법·프로토콜(호흡 O/X → CPR / 음성설명) 흐름.
- Claude 연동: 병력 기반 프로토콜 우선순위, 주변인 음성 설명 처리.
- 두 역할 화면을 오가며 **한 편의 시나리오**로 시연.

**Out (지금은 후순위 / 목업·수동 트리거로 대체):**
- 실제 홈캠 연동 — **폰 고정 촬영**으로 대체, 탐지는 팀원 로컬 구현과 연결(또는 데모용 수동 트리거).
- 실제 백엔드 서버 — **맥북 로컬 처리**로 대체.
- 119 실제 자동 신고 연계, 소방대 실시간 전달 — **시연용 모의(mock)** 로 표현.
- 윤리·동의(영상 촬영 프라이버시) 정식 처리 — **추후 보강** (데모엔 동의 문구/화면만).

## 데모 시나리오 (심사용 — 별도 발전 예정)
어르신 쓰러짐 → 15초 알림 무반응 → 119 자동신고 표시 → 이웃 폰 알림 → 이웃이 달려가 프로토콜(호흡 X → CPR) 수행 → (모의) 소방대 전달. → [`02-design/`](../02-design/README.md)에서 화면 단위로 구체화.

## 확정된 제약 (도메인 공통)
- 데모 우선 — 완벽함보다 동작하는 것.
- 의료 안전성: **응급(호흡·심정지)은 119를 최우선**, 확정 진단·처방 금지, 진단성 출력에 디스클레이머.
- 실제 환자/개인정보 금지, **합성 데이터만.**

## 결정해야 할 것 (주제 확정 시 채우기)
- **목표:** 무엇을, 왜? 어떤 문제를 푸나?
- **주요 사용자:** 누구?
- **핵심 플로우(해피 패스):** 입력 → 처리 → 출력
- **스코프(MVP In/Out):** 데모에 반드시 필요한 것 vs 후순위

## 확정된 제약 (도메인 공통 — 주제와 무관하게 유지)
- 데모 우선 — 완벽함보다 동작하는 것.
- 의료 안전성 규칙 준수 (디스클레이머, 응급 안내, 확정 진단 금지). → 각 스펙 문서의 "의료 안전성 체크" 참고.
- 실제 환자/개인정보 금지, 합성 데이터만.

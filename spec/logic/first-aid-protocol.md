# 응급처치 프로토콜 · 트리아지 스키마 (일반인용)

- **Status:** In Progress

- **관련 코드:** `lib/first-aid/schema.ts`, `lib/first-aid/protocols.ts`, `lib/first-aid/triage.ts`

## 1. 목적 / 사용자 가치
홈캠 AI가 "사람이 쓰러짐"을 감지하면 → 주변인에게 자동 연락 → **환자 상태에 맞는 응급처치를 일반인(비전문가)이 따라 할 수 있게 단계별로 안내**한다. 강원의 급성심정지 생존율 7.2%(전국 최저)·일반인 CPR 시행률 도·군 19.9%(대도시 48.2%의 절반 이하)라는 "목격자 처치 공백"을 직접 겨냥한다. → 근거: [`../../screening/03-ems-dispatch-problems.md`](../../screening/03-ems-dispatch-problems.md), [`04-ems-bottleneck-deepdive.md`](../../screening/04-ems-bottleneck-deepdive.md)

## 2. 근거 가이드라인 (공신력 출처)
프로토콜의 모든 임상 내용은 아래에서 도출한다. 각 프로토콜 객체는 `source[]`로 출처를 명시한다.
- **대한심폐소생협회(KACPR) 「2020 한국심폐소생술 가이드라인」** — 국내 1차 표준 (일반인 가슴압박소생술 권고 포함)
- **American Heart Association(AHA) 2020 Guidelines for CPR & ECC** (2025 focused update 포함) — 국제 교차 참조
- **ILCOR CoSTR** — 국제 합의
- **소방청 119 구급상황관리센터 전화도움 심폐소생술(DA-CPR)** 절차 — 목격자 유도 흐름
- **대한적십자사 응급처치법** — 일반인 교육 표준

> ⚠️ 스키마의 임상 수치(압박 깊이·속도 등)는 위 가이드라인 개정 시 갱신한다. 코드 주석·`source`에 판(edition)·연도를 남긴다.

## 3. 동작 명세 (What)
- 입력: 홈캠 감지 트리거 → 목격자가 앱에서 답하는 트리아지 질문(반응 유무, 호흡 유무, 주요 증상).
- 처리: `triage.ts` 결정트리로 상황을 분류 → 해당 `FirstAidProtocol` 선택 → 단계(step) 순차 안내(음성+큰 글씨+애니메이션 키). 타이머·반복(가슴압박) 지원.
- 출력: 단계별 지시 + 금기(doNot) + 119 신고 문구/위치 + 구급대 인계 요약(handoff).
- 상태: 로딩/오프라인(핵심 프로토콜은 로컬 번들로 네트워크 없이 동작해야 함)/완료(구급대 인계).

## 4. 트리아지 결정트리 (진입 = "쓰러진 사람 발견")
```
0. 현장 안전 확인 (차량·전기·화재·추락 위험 없는지)
1. 반응 확인: 양 어깨 두드리며 큰 소리로 "괜찮으세요?"
   ├─ 무반응 ──▶ 119 신고 + AED 요청(주변인 특정 지목)
   │             └─ 호흡 확인 (10초, 가슴·배 오르내림; 헐떡임=비정상)
   │                 ├─ 무호흡/비정상 ──▶ [P-CPR] 심폐소생술 → 도착 시 [P-AED]
   │                 └─ 정상 호흡 ──▶ 눈에 보이는 위중 징후 확인 (무반응이라 회복자세 전 분기)
   │                        ├─ 전신 경련·떨림      ──▶ [P-SEIZURE] 발작
   │                        ├─ 멈추지 않는 심한 출혈 ──▶ [P-BLEED] 지혈
   │                        └─ 특별한 징후 없음     ──▶ [P-RECOVERY] 회복자세 + 관찰
   └─ 반응 있음 ──▶ 주 증상 분기
        ├─ 목 움켜쥠·기침 못함·질식     ──▶ [P-CHOKING] 기도폐쇄
        ├─ 얼굴 처짐/한팔 마비/말 어눌   ──▶ [P-STROKE] 뇌졸중 FAST
        ├─ 멈추지 않는 심한 출혈         ──▶ [P-BLEED] 지혈
        ├─ 전신 경련/발작                ──▶ [P-SEIZURE] 발작
        └─ 어지러움·일시적 실신          ──▶ [P-SYNCOPE] 실신
```

## 5. 로직 / 규칙 (How) — 안전 원칙
- **119가 최우선.** 무반응 판단 즉시 119 신고를 CPR보다 앞 단계에 배치. AED 동시 요청.
- **위험도 판정·분기는 결정적 데이터(triage.ts)로 수행**, LLM에 위험도 판정을 위임하지 않는다(마음날씨 앱과 동일 원칙). LLM은 안내 문구 자연화·질의응답 보조에만.
- **일반인 기본값 = 가슴압박소생술(hands-only).** 인공호흡은 훈련·의지가 있는 경우의 선택지(30:2)로만 표기.
- **비정상 호흡(agonal gasping)은 정상 호흡이 아니다** → 심정지로 간주하고 CPR.
- **대상 구분**: 성인 / 소아(1세~사춘기) / 영아(1세 미만) — 압박 깊이·방법·AED 패드가 다르므로 프로토콜에 `patientType` 명시.
- **금기(doNot)** 필드 필수: 예) 발작 중 입에 물건 넣지 않기, 박힌 이물질 빼지 않기.
- **과잉 확신 금지**: 확정 진단 표현 금지("심장마비입니다" ✕ → "가슴압박이 필요한 상태로 보여요").

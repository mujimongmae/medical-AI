- **Status:** Draft

이 폴더는 UI/UX의 진실 원천이다. 화면을 만들거나 바꾸기 전에 여기부터 수정한다.

> ✅ **아이디어 확정** ([`../00-overview/`](../00-overview/README.md)): 마을 공동체 응급대응 앱(환자/이웃 역할 모드). 화면 미작성 — 첫 설계 대기.

## 무엇을 담나
- **화면 설계:** 각 화면의 레이아웃·상태(로딩/에러/빈/성공)·인터랙션
- **디자인 토큰:** 색상(고대비)·타이포·간격·반경
- **접근성:** WCAG AA (터치 타겟 ≥ 44px, 명도 대비 ≥ 4.5:1)
- **컴포넌트:** 재사용 UI 규격

### 역할 선택/등록 (`app/src/App.tsx` RolePicker)
- 큰 이름 입력(text-xl) + **보호자 대신 입력** 안내 문구. placeholder에 "(합성)" 명시.
- 역할 버튼 2개(환자/이웃): 초대형(py-6), 역할별 한 줄 설명 부제 포함.
- 등록 실패(브로커 미연결) → `alert()` 대신 **인라인 에러 문구**(빨강) + 재시도 안내. 등록 중 "등록 중…" 표시.
- 상태: 로딩(busy)·에러 인라인 표시로 통일.

### 환자 화면 (`app/src/roles/PatientView.tsx`)
- **평상시:** ws 접속 상태 표시 — 연결됨 "● 정상 감시 중"(safe), 미연결 "● 연결 대기 중…"(yellow) + 안내. 데모 트리거 버튼은 로딩/에러 상태 포함.
- **15초 알림:** 전체화면 고대비(danger 배경), 초대형 원형 카운트다운 숫자(text-7xl), 반복 알림음(880Hz)+진동, 초대형 흰색 "괜찮아요(해제)" 버튼. 0초 도달 시 safe 배경으로 전환 + "도움을 요청했습니다" 안내(요청 완료 상태).

### 전역 (`app/src/index.css`)
- 버튼 최소 56px, `focus-visible` 고대비 링(키보드/보조기기), disabled 커서/투명도.
- 한 폰 이중 테스트: "역할 변경 (환자 ↔ 이웃)" 버튼 + 실수 방지 confirm.

## 전역 원칙 (도메인 공통)
- 모바일 우선, 한 화면 한 목적.
- 응급/중요 안전 정보는 항상 최상단·고대비.
- 모든 진단성 화면에 디스클레이머 고정 노출.

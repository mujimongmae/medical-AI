## 1. 목적
심사위원 앞에서 **"강원 독거노인 마을 공동체 응급대응"** 핵심 가치를 한 편의 시나리오로 시연한다:
쓰러짐 감지(모의) → 15초 본인 확인 → 무반응 시 **119 자동신고(모의) + 가까운 이웃 호출** → 이웃이 달려가 **결정트리 트리아지 → CPR 프로토콜(메트로놈)** 수행 → **음성 상황설명(구급대 전달용 요약)**. 완전한 앱보다 **눈으로 확인되는 흐름**이 최우선.

> ⚠️ 전제(스펙 준수): 실제 영상인식·119·소방 연계는 **모의(mock)**. 데이터는 **합성만**. 응급은 **119 최우선**, 확정 진단·처방 없음, 디스클레이머 상시 노출.

## 2. 준비물 (Setup)
| 항목 | 내용 |
|------|------|
| 맥북(브로커) | `npm run dev:all` → 앱(Vite :5173) + 브로커(Fastify :8787) 동시 기동. 콘솔 로그 = 심사 보조 화면(119 모의·이웃 선별 로그가 보임) |
| 폰 A (환자) | 어르신 역할. 큰 화면·소리 확인 |
| 폰 B~D (이웃) | 이웃 역할 1~3대(최소 1대). 같은 마을 `방림리`로 자동 등록됨 |
| 네트워크 | **HTTPS 터널 필수** — `cloudflared tunnel --url http://localhost:5173`(또는 `ngrok http 5173`) → 발급 URL을 폰에서 접속. 마이크(getUserMedia)·오디오 자동재생은 HTTPS(보안 컨텍스트)에서만 동작 |
| 백업 | AP 격리 대비 **맥북+폰 모두 폰 핫스팟**에 연결. 폰 1대만 있으면 화면 하단 "역할 변경 (환자 ↔ 이웃)"으로 번갈아 시연 가능 |

**사전 점검(리허설):** 오디오는 사용자 제스처 후 재생됨 → 폰에서 화면을 한 번 터치해 두면 15초 알림음·메트로놈이 확실히 난다. 볼륨 최대. `npm run typecheck`·`npm run build` 통과 확인.

**Claude 연동(선택):** `ANTHROPIC_API_KEY`가 **맥북 서버 환경(.env.local)** 에 있으면 음성 요약이 Claude로 자연화되고, 없으면 자동으로 에코 폴백(“현장 전언: …”)이 나온다. **키가 없어도 데모는 깨지지 않는다.** 키는 절대 앱/클라이언트에 넣지 않는다(서버 전용).

## 3. 등장 인물 (합성 데이터)
- 서버 시작 시 시드(`server/seed.ts`)가 마을 `방림리`에 **환자 김복순(합성, 고혈압·협심증) + 이웃 4명(합성)** 을 자동 주입한다. 전부 가짜.
- 폰으로 접속해 등록하는 사용자도 자동으로 `방림리` 소속이 된다(같은 마을 → 이웃 호출 대상).

## 4. 시연 순서 (해피 패스, 약 4~5분)

### 0) 도입 (30초)
- 한 줄 소개: "강원은 병원이 멀어 골든타임을 놓칩니다. 우리는 **발견 지연**과 **초기 대처 공백** 두 지점을 마을 공동체로 메웁니다." (근거: 농촌 OHCA 생존 3.4% vs 도시 8.7%, 강원 급성심정지 생존율 7.2% 전국 최저 — `screening/`)
- 폰 A(환자)·폰 B(이웃) 화면을 나란히 보여준다. 두 화면 모두 **상단 디스클레이머** 상시 노출을 짚는다.

### 1) 등록 (30초)
- 폰 A: 이름 입력(placeholder "예: 김순자 (합성)") → **"어르신 (환자)"** 선택. → 평상시 화면 **"● 정상 감시 중"**(연결 상태 표시).
- 폰 B: 이름 입력 → **"이웃 (도움 주는 사람)"** 선택. → **"● 대기 중"**.
- 멘트: "설치는 하나, 역할만 다릅니다. 어르신은 폰만 켜두면 됩니다."

### 2) 쓰러짐 발생 → 15초 본인 확인 (40초)
- 폰 A에서 **`[데모] 쓰러짐 발생 시뮬레이션`** 버튼을 누른다. (실서비스에선 이 자리를 맥북 로컬 영상인식의 `POST /fall-event`가 대체 — "교체 가능한 이벤트 소스")
- 폰 A: **전체화면 빨강 + 초대형 원형 카운트다운(15→0) + 반복 알림음(880Hz)·진동.**
- 멘트: "오탐이면 '괜찮아요'를 눌러 즉시 해제됩니다." (원하면 여기서 한 번 눌러 **해제(SELF_CANCEL)** → 종료를 보여주고, 다시 눌러 재현)
- 이번엔 **누르지 않고 15초 무반응**을 연출.

### 3) 119 자동신고(모의) + 이웃 호출 (30초)
- 0초 도달 → 폰 A 화면이 **안전(초록) 배경 + "도움을 요청했습니다"** 로 전환(요청 완료 상태).
- **맥북 콘솔**을 가리키며: `[MOCK 119] 자동 신고 …` 로그가 **이웃 호출보다 먼저** 찍힘을 보여준다(=응급은 119 최우선). 이어 `이웃 호출 N명 …` 로그.
- 폰 B(이웃): **🚨 응급 호출 — 지금 가주세요** + 환자 카드(이름·위치 "방림리 (합성 주소)"·진입 안내·병력요약).

### 4) 이웃 출동 → 트리아지 (40초)
- 폰 B: **"도착했습니다 — 응급처치 시작"** → 결정트리(`lib/first-aid/triage.ts`) 시작.
- Q1 반응 확인 → **"무반응"** → Q2 호흡 확인 → **"아니오 — 숨을 안 쉬거나 헐떡임"** 선택.
- 멘트: "위험도 분기는 **LLM이 아니라 결정트리**가 판정합니다. 최단 2질문으로 CPR에 도달합니다."

### 5) CPR 프로토콜 + 메트로놈 (50초)
- 자동으로 **P-CPR(매우 위급)** 프로토콜 화면: 단계별 지시 + 대상별(성인/소아/영아) 수치 + **⛔ 하지 마세요** + **🚑 구급대 도착 시 전달**.
- 가슴압박 단계의 **"▶ 박자 시작"** → 분당 110회 박자음·진동 + **30:2 세트 카운트 시각화** + 30회째 "인공호흡 2회 시점(선택)" 하이라이트.
- 하단 **프로토콜 disclaimer + GLOBAL_DISCLAIMER** 를 짚는다. (참고용·전문 처치 대체 아님)

### 6) 음성 상황설명 (40초)
- 같은 화면의 **🎤 상황 설명** 패널 → "말하기 시작"(STT 미지원 기기면 자동으로 텍스트 입력) → 예: "60대 남성, 가슴을 붙잡고 쓰러졌고 지금은 숨을 헐떡입니다." → **"요약 보내기"**.
- 서버(`POST /api/voice` → `server/claude.ts`)가 **1~2문장 객관 요약**(진단·처방 없음) 반환 → 화면 표시. "이 요약을 구급대에 전달합니다(추후 실시간 relay)."
- Claude 키가 없으면 폴백 요약이 나옴을 언급(데모 안정성).

### 7) 마무리 (20초)
- 요약: "발견→신고→초기대처 공백을 **마을 이웃 + 결정트리 + AI 요약**으로 메웁니다. 데모는 맥북 브로커지만, 그대로 **Supabase/Vercel 클라우드로 스왑** 가능합니다."

## 5. 병력 기반 우선순위(선택 심화 시연)
기본 해피패스(폰 A 자체 등록)는 병력이 없어 이웃 카드에 "특이사항 없음"으로 뜬다. **병력 기반 우선순위 힌트**(`NEIGHBOR_ALERT.priorityHint`)를 보여주려면 **시드 환자 김복순(고혈압·협심증)** 으로 이벤트를 낸다:
```
curl -X POST http://localhost:8787/api/fall-event \
  -H 'content-type: application/json' -d '{"patientId":"seed-patient-1"}'
```
- 시드 환자는 실제 폰이 없어 **15초 본인 알림 화면은 뜨지 않지만**, 15초 후 이웃 폰에 **병력요약(고혈압, 협심증) + 심장 우선 힌트(P-CPR>P-AED)** 가 포함된 호출이 간다. (분기 자체는 여전히 클라 `triage.ts` 결정트리)
- 즉, **완전한 환자측 15초 흐름 = 폰 A 자체 트리거**, **풍부한 병력 카드·우선순위 = 시드 환자 curl 트리거**. 둘 중 시나리오에 맞게 선택.

## 6. 실패·엣지 대비 (데모 리스크)
| 리스크 | 대비 |
|--------|------|
| 알림음/메트로놈 무음 | 재생 전 화면 1회 터치(오디오 제스처), 볼륨 최대, HTTPS 접속 |
| 마이크 권한 거부 / iOS WKWebView STT 미지원 | 자동으로 **텍스트 입력 폴백** — 그대로 타이핑해 진행 |
| 이웃 폰 미접속(호출 0명) | 서버는 119 모의만 유지 + 로그 경고. 이웃 폰이 `방림리`로 등록·연결됐는지 확인 |
| ws 끊김 | 앱 자동 재연결(HELLO 재전송). 환자 화면 "● 연결 대기 중…" 표시되면 잠시 대기 |
| 브로커 미기동 | 등록/트리거 시 인라인 에러 문구 노출 → `npm run dev:all` 확인 |
| Claude 키 없음/한도 | 음성 요약이 폴백으로 동작(데모 안 깨짐) |

## 7. 안전·프라이버시 체크 (시연 전 필수)
- [x] 상단 `GLOBAL_DISCLAIMER` 상시 + 프로토콜 화면 프로토콜별 disclaimer 재노출.
- [x] 응급 시 **119 모의신고가 이웃 호출보다 먼저**(서버 `escalate`) — 콘솔 로그로 시연.
- [x] 합성 데이터만(시드·등록·카드 전부 "(합성)"/"(합성 주소)"). 실제 PHI/PII 없음.
- [x] `ANTHROPIC_API_KEY`는 서버 전용(클라 번들·커밋 미노출). 키 없어도 폴백 동작.

## 1. 목적 / 사용자 가치
홈캠(폰 카메라) AI가 "사람이 쓰러짐"을 감지하면 → 이웃(주변인)에게 자동 연락 → **환자 상태에 맞는 응급처치를 일반인(비전문가)이 따라 할 수 있게 단계별로 안내**한다. 강원의 급성심정지 생존율 7.2%(전국 최저)·일반인 CPR 시행률 도·군 19.9%(대도시 48.2%의 절반 이하)라는 "목격자 처치 공백"을 직접 겨냥한다.
→ 근거: [`../../screening/03-ems-dispatch-problems.md`](../../screening/03-ems-dispatch-problems.md), [`../../screening/04-ems-bottleneck-deepdive.md`](../../screening/04-ems-bottleneck-deepdive.md)

## 2. 근거 가이드라인 (공신력 출처)
프로토콜의 모든 임상 내용은 아래에서 도출한다. 각 프로토콜 객체는 `source[]`로 출처를 명시한다.
- **대한심폐소생협회(KACPR) 「2020 한국심폐소생술 가이드라인」** — 국내 1차 표준 (일반인 가슴압박소생술 권고 포함)
- **American Heart Association(AHA) 2020 Guidelines for CPR & ECC** (2025 focused update 포함) — 국제 교차 참조
- **ILCOR CoSTR** — 국제 합의
- **소방청 119 구급상황관리센터 전화도움 심폐소생술(DA-CPR)** 절차 — 목격자 유도 흐름
- **대한적십자사 응급처치법** — 일반인 교육 표준

> ⚠️ 스키마의 임상 수치(압박 깊이·속도 등)는 위 가이드라인 개정 시 갱신한다. 코드 주석·`source`에 판(edition)·연도를 남긴다.

## 3. 동작 명세 (What)
- 입력: 홈캠 감지 트리거 → 이웃이 앱에서 답하는 트리아지 질문(반응 유무, 호흡 유무, 주요 증상).
- 처리: `triage.ts` 결정트리로 상황을 분류 → 해당 `FirstAidProtocol` 선택 → 단계(step) 순차 안내(음성+큰 글씨+애니메이션 키). 타이머·반복(가슴압박) 지원.
- 출력: 단계별 지시 + 금기(doNot) + 구급대·구급상황실 전달 정보(증상·발생 시각) + 구급대 인계 요약(handoff). **(119 신고는 자동 전제 — 아래 §4)**
- 상태: 로딩/오프라인(핵심 프로토콜은 로컬 번들로 네트워크 없이 동작해야 함)/완료(구급대 인계).

## 4. 트리아지 결정트리 (진입 = 홈캠이 "쓰러짐" 감지, 코드: `triage.ts`, 루트 `TRIAGE_ROOT="Q1"`)
**전제: 홈캠 감지 시 119 신고는 자동으로 이뤄진다.** 따라서 트리아지에 "신고하세요" 단계를 두지 않고, **CPR 판단까지 최소 질문(반응→호흡, 2단계)**으로 도달한다. CPR 상황이 아니면 목격자가 환자에게 **질문**해 상태를 파악·연결한다. 현장 안전 확인은 질문 노드가 아니라 **첫 화면 상시 안내**로 제공.
```
Q1. 반응 확인: 어깨 두드리며 크게 불러 반응(눈뜸·움직임·대답)이 있나?
   ├─ 무반응 ──▶ Q2. 호흡 확인 (10초; 헐떡임=비정상, 애매하면 '없음')
   │              ├─ 정상호흡 없음/헐떡임 ──▶ [P-CPR] → AED 도착 시 [P-AED]   ★최단 경로(질문 2개)
   │              └─ 정상호흡 있음 ──▶ Q3. 눈에 보이는 위중 징후?
   │                       ├─ 전신 경련·떨림      ──▶ [P-SEIZURE]
   │                       ├─ 멈추지 않는 심한 출혈 ──▶ [P-BLEED]
   │                       └─ 없음                ──▶ [P-RECOVERY]
   └─ 반응 있음 ──▶ S1. 환자에게 물어 주 증상 파악
        ├─ 목 움켜쥠·기침/말 못함(질식)  ──▶ [P-CHOKING]
        ├─ 얼굴 처짐/한팔 마비/말 어눌     ──▶ [P-STROKE]
        ├─ 멈추지 않는 심한 출혈          ──▶ [P-BLEED]
        ├─ 잠깐 정신 잃음·어지러움(실신)  ──▶ [P-SYNCOPE]
        └─ 해당 없음/모르겠음            ──▶ S1-observe: 안정·관찰 (상태 변화 시 Q1 재확인)
```
> 무반응인데 숨은 쉬는 경우(Q3)는 회복자세로 보내기 전에 경련·대량출혈을 먼저 걸러 해당 프로토콜에 도달시킨다(감수 반영). 발작은 활성 경련 중 무반응이므로 Q3 경로에 둔다.

## 5. 프로토콜 목록 (8종, 코드: `protocols.ts`)
| id | 이름 | 응급도 | 적용 상황 |
|----|------|--------|-----------|
| `P-CPR` | 심폐소생술(가슴압박) | critical | 무반응 + 무호흡/비정상호흡 |
| `P-AED` | 자동심장충격기(AED) 사용 | critical | 심정지 환자에게 AED 도착 시 |
| `P-CHOKING` | 기도폐쇄(질식) 응급처치 | critical | 이물질로 기도 막힘 |
| `P-STROKE` | 뇌졸중 의심(FAST) | critical | 갑작스런 얼굴·팔·말 이상 |
| `P-BLEED` | 심한 출혈 지혈 | critical | 멈추지 않는/솟구치는 출혈 |
| `P-SEIZURE` | 발작·경련 대응 | urgent | 전신 경직·떨림 + 의식 없음 |
| `P-RECOVERY` | 회복자세 | urgent | 무반응 + 정상 호흡 |
| `P-SYNCOPE` | 실신·기절 대응 | caution | 잠깐 정신 잃고 곧 회복 |

## 6. 로직 / 규칙 (How) — 안전 원칙
- **119 신고는 홈캠 감지 시 자동 수행을 가정.** 트리아지는 신고 다이얼 단계 대신 (1) AED 확보·역할 분담, (2) 구급대·구급상황실 전달 정보(증상·발생 시각)를 안내. 현장 안전 확인은 질문 노드가 아닌 첫 화면 상시 안내.
- **CPR까지 최소 단계.** 무반응→호흡 확인→(없으면) 즉시 가슴압박. 애매하면 시행 쪽으로(의심되면 시행).
- **위험도 판정·분기는 결정적 데이터(`triage.ts`)로 수행**, LLM에 위험도 판정을 위임하지 않는다. LLM은 안내 문구 자연화·질의응답 보조에만.
- **일반인 기본값 = 가슴압박소생술(hands-only).** 인공호흡은 훈련·의지가 있는 경우의 선택지(30:2)로만 표기.
- **비정상 호흡(agonal gasping)은 정상 호흡이 아니다** → 심정지로 간주하고 CPR.
- **대상 구분**: 성인 / 소아(1세~사춘기) / 영아(1세 미만) — 압박 깊이·방법·AED 패드가 다르므로 프로토콜에 `patientType` 명시.
- **금기(doNot)** 필드 필수: 예) 발작 중 입에 물건 넣지 않기, 박힌 이물질 빼지 않기.
- **과잉 확신 금지**: 확정 진단 표현 금지("심장마비입니다" ✕ → "가슴압박이 필요한 상태로 보여요").

## 7. 데이터 / 스키마 (코드: `schema.ts`)
- `schema.ts`: `FirstAidProtocol`, `ProtocolStep`, `TriageNode`, `GuidelineRef`, `Urgency`, `PatientType`, `RepeatRule` 타입 + `GLOBAL_DISCLAIMER`.
- `protocols.ts`: 위 8개 프로토콜 데이터(스키마 준수).
- `triage.ts`: §4 결정트리 데이터(`TRIAGE`, 진입 `TRIAGE_ROOT="Q1"`).
- PHI/PII 없음 — 순수 지식 콘텐츠. **오프라인 번들 가능해야 함**(응급 상황에 네트워크 의존 위험).

## 8. 아키텍처 연동 (이 데모 구조에서)
- 트리아지 응답은 [`01-messaging-protocol.md`](./01-messaging-protocol.md)의 `PROTOCOL_ANSWER`로 서버에 전달되고, 다음 단계는 `PROTOCOL_STEP`으로 이웃앱에 push된다.
- 환자 **병력 기반 우선순위**: 서버가 환자 병력을 참고해 Claude로 프로토콜/문구 우선순위를 정렬(위험도 분기 자체는 `triage.ts` 결정트리 유지).
- **오프라인 대비**: 8개 프로토콜·트리아지는 클라이언트 로컬 번들로도 동작(네트워크 끊겨도 단계 안내 지속). Claude 보조는 온라인일 때만.

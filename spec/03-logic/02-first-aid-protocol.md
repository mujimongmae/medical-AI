## 1. 목적 / 사용자 가치
홈캠 AI가 쓰러짐을 감지 → 이웃에게 자동 연락 → **비전문가 이웃이 따라 할 수 있게** 응급처치를 안내한다. 강원 급성심정지 생존율 7.2%(전국 최저)·일반인 CPR 시행률 도·군 19.9%의 "목격자 처치 공백"을 겨냥.

## 2. 근거 가이드라인 (공신력 출처)
프로토콜 임상 내용의 출처. 각 프로토콜 객체는 `source[]`로 명시.
- **대한심폐소생협회(KACPR) 「2020 한국심폐소생술 가이드라인」** — 국내 1차 표준(일반인 가슴압박소생술)
- **AHA 2020 Guidelines for CPR & ECC**(2025 focused update) — 국제 교차참조
- **ILCOR CoSTR** · **소방청 119 DA-CPR** · **대한적십자사 응급처치법**
> ⚠️ 임상 수치는 가이드라인 개정 시 갱신. `source`에 판·연도 기록.

## 3. 동작 명세 (What) — 2갈래 흐름
진입: 쓰러짐 감지 → **119 자동 신고** → 가까운 이웃 3~4명 호출 → 이웃 도착.
그다음 **딱 하나만 먼저 가른다: "지금 CPR을 해야 하나?"**
- **(A) CPR 경로** — 의식·호흡이 **둘 다 없음** → 즉시 가슴압박 화면(시각자료 중심).
- **(B) AI 음성 추천 경로** — 의식이나 호흡이 **있음** → 이웃이 환자 상태를 **말로 설명** → STT → **Claude가 병력+증상 종합** → **가장 가능성 높은 상태 + 추천 응급처치**를 큰 글씨로 제시.

상태: 로딩 / 오프라인(**CPR 경로는 로컬 번들로 항상 동작**, AI 추천은 온라인일 때) / 완료(구급대 인계).

**UX 원칙(주 사용자 = 어르신):** 한 화면에 **한 메시지**. 작은 글자 많이 X → **큰 글씨 + 큰 시각자료(GIF·그림)**, 텍스트 최소. 세부 수치는 보조로 작게.

## 4. 트리아지 (진입 = 이웃 도착, 코드: `triage.ts`)
```
Q1. 의식(반응) 있나요?  — 어깨를 두드리며 크게 불러본다
   ├─ 있음 ─────────────────────────▶ (B) AI 음성 추천
   └─ 없음 → Q2. 숨을 정상적으로 쉬나요? (10초, 헐떡임=비정상=아니오)
             ├─ 아니오(무호흡/헐떡임) ──▶ (A) CPR 화면   ★질문 2개로 도달
             └─ 예(정상 호흡) ─────────▶ (B) AI 음성 추천
```
- **CPR 조건 = 의식 없음 AND 호흡 없음.** 그 외 전부 (B).
- 현장 안전 확인은 질문 노드가 아니라 **첫 화면 상시 안내**.

### 4.1 (A) CPR 화면 — 의식·호흡 둘 다 없음
- 큰 문구: **"가슴압박을 시작하세요"** (보조: "훈련돼 있으면 30회마다 인공호흡 2회")
- **CPR GIF (분당 100~120회 리듬)** + **압박 위치 그림**(가슴 정중앙, 복장뼈 아래쪽 절반)
- **메트로놈**(분당 110회 박자음 + 진동, 세트 카운트) — 이미 구현된 것 활용
- 보조(작게): 성인 약 5cm·소아 4~5cm·영아 4cm / 중단 10초 이내 / AED 도착·움직임·정상호흡·구급대까지 지속
- 하면 안 됨: 헐떡임을 호흡으로 오인해 미루기 / 명치·갈비뼈 압박 / 인공호흡 하느라 압박 오래 멈추기
- AED 도착 시 → `P-AED` 화면(패드 위치 그림 + 음성안내 따르기)

### 4.2 (B) AI 음성 추천 화면 — 의식이나 호흡이 있음
1. 큰 버튼: **"환자 상태를 말로 설명해 주세요"** → 녹음 시작.
2. **온디바이스 STT(ko-KR)** 로 텍스트화 → 서버 `POST /api/voice { transcript }`.
3. 서버가 **환자 병력(사전 입력) + 증상 설명**을 Claude에 전달 → Claude가 종합:
   - **가장 가능성 높은 상태 1개**(확정 아님, "…가능성이 높아요" 표현)
   - **그에 맞는 추천 응급처치**(짧고 실행 가능한 문장, 어르신용 큰 글씨)
   - 매칭되는 지식베이스 프로토콜 id(있으면) → 상세 화면 연결
4. 예: "피가 많이 나요/팔이 잘렸어요/얼굴이 파래졌어요" → 지혈 / 절단부 처치 / 질식 처치를 추천.
5. 화면 하단 고정: **디스클레이머 + "119 지시 우선"**.

> **왜 AI인가**: 증상 조합·병력 반영은 고정 O/X 트리로 다 못 담음. AI가 자유 발화를 종합해 **가장 적절한 처치 하나**를 큰 글씨로 제시 → 어르신이 헤매지 않음.

## 5. 프로토콜 지식베이스 (8종, 코드: `protocols.ts`) — AI 추천의 근거 + 상세 화면
AI 추천은 아래 8종 지식베이스에 근거해 매핑한다. `P-CPR`/`P-AED`는 CPR 경로가 직접 사용.

| id | 이름 | 응급도 | 적용 |
|----|------|--------|------|
| `P-CPR` | 심폐소생술(가슴압박) | critical | 무의식+무호흡 → (A) 경로 |
| `P-AED` | 자동심장충격기(AED) | critical | 심정지 + AED 도착 |
| `P-CHOKING` | 기도폐쇄(질식) | critical | 이물질로 숨 못 쉼 |
| `P-STROKE` | 뇌졸중(FAST) | critical | 얼굴·팔·말 이상 |
| `P-BLEED` | 심한 출혈 지혈 | critical | 멈추지 않는 출혈·절단 |
| `P-SEIZURE` | 발작·경련 | urgent | 전신 경련 |
| `P-RECOVERY` | 회복자세 | urgent | 무의식+정상호흡 |
| `P-SYNCOPE` | 실신 | caution | 잠깐 정신 잃고 회복 |

각 프로토콜은 `steps`·`doNot`·`handoff`·`disclaimer`·`byPatient(성인/소아/영아)` 보유. (상세 임상 내용은 코드가 진실 원천)

## 6. 로직 / 규칙 (How) — 안전 원칙 (개정)
- **생명 직결 결정(CPR 여부)은 규칙 기반**: 의식·호흡 2개로만 판단, **LLM에 위임 안 함**. 헐떡임=심정지로 간주, 애매하면 시행. 일반인 기본 = 가슴압박(hands-only).
- **그 외 상태 판단·처치 추천은 AI 보조**(경로 B): 단
  - **확정 진단 금지** → "…일 가능성이 높아요" 식 표현.
  - **119/구급대 지시 우선**, 환자 **병력 반영**, **디스클레이머** 고정 노출.
  - AI는 지식베이스(8종) 범위 내에서 추천, 위험/불명확하면 "구급상황실(119)에 물어보세요"로 안내.
- **대상 구분**(성인/소아/영아) 수치는 프로토콜 `byPatient`로.
- **어르신 UX**: 한 화면 한 메시지, 큰 글씨·큰 시각자료, 음성 우선.

## 7. 데이터 / 스키마 (코드)
- `schema.ts`: `FirstAidProtocol`,`ProtocolStep`,`TriageNode`,`GuidelineRef`,`Urgency`,`PatientType`,`RepeatRule` + `GLOBAL_DISCLAIMER`.
- `triage.ts`: §4 결정트리(의식→호흡→CPR/AI).
- `protocols.ts`: 8종 지식베이스.
- `server/claude.ts`: (B) 경로의 병력+증상 종합·추천 로직(구조화 출력).
- PHI/PII 없음(합성만). CPR 경로는 **오프라인 번들** 가능해야 함.

## 8. 아키텍처 연동
- (A) CPR: 전부 클라이언트 로컬(오프라인 OK) + 메트로놈.
- (B) AI: 음성→STT(온디바이스)→`POST /api/voice`→Claude→추천 반환→큰 글씨 표시. (spec 01 §2.2)
- 환자 병력은 등록 시 입력, 서버가 이벤트의 환자 병력을 Claude 컨텍스트로 사용.

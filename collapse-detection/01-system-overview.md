# 01 — 시스템 개요 & 내 파트 경계

- **Status:** Draft
- **Owner:** 김준휘 (감지 파트)
- **Last updated:** 2026-07-02

## 1. 제품 한 줄
홈캠으로 심뇌혈관 응급(쓰러짐)을 감지해 **무목격·독거 상황의 신고 지연을 분 단위로 줄이는** 초동 대처 앱.

## 2. 왜 (근거)
- 병원 밖 심정지의 **47%가 가정 발생**, 목격 시 생존 2.61배 — 즉시 인지가 생존 분기점 ([screening/06](../screening/06-auto-alert-efficacy.md))
- 심정지 1분 지연당 생존율 **7~10%p 감소** → "몇 분 빨리 신고했나"가 결과를 지배
- 강원 등 의료취약지·고독사 증가 맥락에서 "쓰러졌는데 아무도 모름"을 겨냥

## 3. 전체 플로우 (해피 패스)
```
[홈캠=핸드폰] 쓰러짐 감지(내 파트)
    → emergency_events insert (후보 발행) ── 내 경계 여기까지
        → [앱] 본인 폰에서 큰 알람 "정말 쓰러지셨나요? [취소]"
            → 무반응 N초 → 119 자동 연결 + 주변인 연락 (팀원 파트)
            → 취소 → 종료 (오탐 회수)
```
**핵심**: 최종 취소 게이트는 **앱**에 있음. 감지는 "후보"만 발행 → 오탐이어도 본인이 앱에서 끄면 됨 (Apple Watch 패턴).

## 4. 주요 사용자
- 대상자: 독거·고령 등 가정 내 1인 (홈캠 관찰 대상)
- 보호자·주변인: 알림 수신·연락 대상
- (다운스트림) 119: 최종 신고 대상 — 데모는 시뮬레이션

## 5. MVP 스코프
**In (반드시 데모):**
- 핸드폰 카메라 → 노트북 홈캠 창 실시간 표시
- COCO-SSD 객체 박스 + 라벨 (사람·침대·소파 등)
- MoveNet 포즈 → **급격한 전이(쓰러짐) 감지**
- 상태 배너 "정상 / 쓰러짐 의심 / 쓰러짐!"
- `emergency_events` insert → 앱 창이 Realtime으로 수신·반응

**Out / Stretch:**
- rPPG 심박·호흡, agonal breathing 오디오 → Stretch(또는 "실험적" 라벨/시뮬)
- Claude Vision 2차 확인 → Stretch
- 실제 119 API·문자 발송 → 시뮬레이션
- 앱 네이티브 캡핑(Capacitor) → 차후

## 6. 크리틱 반영 (설계 결정)
- **급격한 움직임 1차** — 데모 동작(의자→급락)과 판정축 일치. Zone·무동작은 보조.
- **영상 링크: 폰=웹캠 기본**, WebRTC는 "되면 좋고" (데모 안정성 우선)
- **Zone: 수동 지정** 기본 (COCO 자동 침대탐지는 보너스)
- **포지셔닝**: "심정지 진단"❌ → "쓰러짐·부동 감지 후 신고 시간 단축"⭕

## 7. 의료 안전성
- 진단 아님 — "징후 감지 + 신고 지원". 확진은 출동 구급대·병원.
- 디스클레이머 필수: "본 정보는 참고용이며 의학적 진단이 아닙니다."
- 페일세이프: 놓치는 것보다 오탐이 안전 (단, 앱 취소 게이트로 오탐 회수)
- 합성/데모 데이터만. 실제 인물 영상 미저장.

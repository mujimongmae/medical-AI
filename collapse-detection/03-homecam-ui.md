## 1. 목적 / 사용자 가치
노트북에 띄우는 **관찰용 홈캠 창**. iPhone 연속성 카메라 영상 + 실시간 오버레이(박스·스켈레톤) + 상태 배너를 한 화면에. 감지 상태를 사람이 즉시 이해하고, 문제 시 원인을 알 수 있게 한다.

## 2. 화면 구성 (레이아웃)
```
┌───────────────────────────────────────────────┐
│  ● 상태 배너 (풀폭, 상단 고정)                  │  ← 최상위, 색+아이콘+텍스트
├───────────────────────────────────────────────┤
│                                                │
│   [ 비디오 + Canvas 오버레이 겹침 ]             │  ← 16:9, 중앙
│     - COCO 박스 + 라벨                          │
│     - MoveNet 스켈레톤                          │
│     - Zone 사각형 (지정 시)                     │
│                                                │
├───────────────────────────────────────────────┤
│  하단 바: fps · 모델상태 · 카메라 선택 · Zone편집│
└───────────────────────────────────────────────┘
```
- 오버레이는 `<video>` 위 `<canvas>` 절대배치, 좌표계 = 비디오 표시 크기에 스케일.
- 고령 관찰자 배려: 배너 큰 폰트/고대비(CLAUDE.md 접근성).

## 3. 상태별 표시 (상태머신 §02.3 매핑)
| 상태 | 배너 색/문구 | 오버레이 |
|---|---|---|
| NORMAL | 초록 · "정상 관찰 중" | 박스·스켈레톤 기본색 |
| SUSPECT | 노랑 · "쓰러짐 의심 — 확인 중" | 대상 박스 노랑 강조 + 카운트다운(`T_CONFIRM`) |
| CANDIDATE | 빨강(점멸) · "쓰러짐 감지! 앱에서 확인 중" | 대상 박스 빨강 + 발행 뱃지 |
| CANCELLED | 회색 · "취소됨 (오탐 회수)" 수초 후 NORMAL | 기본색 복귀 |
| ESCALATED | 진빨강 · "119 연결 진행" (앱 상태 반영) | 빨강 유지 |
| (공통) | 하단 소형 디스클레이머 상시 | — |

- 상태 전이는 부드러운 색 트랜지션(급변 눈부심 방지, 단 CANDIDATE 점멸은 유지).
- **디스클레이머 상시:** "본 정보는 참고용이며 의학적 진단이 아닙니다."

## 4. iPhone 연속성 카메라 연결 안내
연속성 카메라 = macOS가 iPhone을 **일반 웹캠**으로 노출 → `getUserMedia` 로 그대로 잡힘(WebRTC 페어링 불필요).
- **연결 조건 안내 카드(첫 진입/미검출 시):**
  - 같은 Apple ID · Wi-Fi/Bluetooth 켜기 · iPhone 근처 거치
  - iPhone 잠금/가로 거치 시 카메라로 노출됨
- **카메라 선택 UI:** `enumerateDevices` 로 videoinput 목록 → iPhone(연속성) 항목 선택. 폴백: 내장/USB 웹캠.
- **가이드 문구:** "iPhone이 목록에 없으면: 잠금 해제 후 거치대에 두고 새로고침."

## 5. 로딩 / 에러 / 권한 상태 (데모 필수 — 예외처리 안 하면 깨짐)
| 상태 | 조건 | UI |
|---|---|---|
| 권한 요청 | 카메라 미허용 | "카메라 접근을 허용해 주세요" + 재요청 버튼 |
| 권한 거부 | `NotAllowedError` | 브라우저 설정 안내 + 재시도 |
| 카메라 없음 | videoinput 0개 | 연속성 안내 카드(§4) 노출 |
| 모델 로딩 | TF.js/COCO/MoveNet 다운로드 중 | 스켈레톤 스피너 + "모델 로딩 중… (최초 1회)" |
| 모델 실패 | 로드/추론 예외 | "감지 엔진 오류 — 새로고침" (침묵 금지, §02.5) |
| 추적 끊김 | 대상 소실 | 배너에 "추적 끊김" 배지(상태는 유지) |
| 저프레임 | fps < 임계 | 하단 바 경고색(감지 지연 가능 고지) |

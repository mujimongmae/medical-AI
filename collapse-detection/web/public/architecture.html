🏠 홈캠 쓰러짐 감지 — 전체 구조 & 모델 설명

카메라에 잡힌 사람이 쓰러지면, 여러 AI 레이어가 단계적으로 판단해 신고 지연을 분 단위로 줄입니다.

SDOH 기반3층 캐스케이드 온디바이스 + 로컬 모델 + VLM

0. 왜 이걸 만드나 (근거)

47%병원 밖 심정지가 가정에서 발생 (질병청 2023)
7–10%p심정지 1분 지연당 생존율 감소 (Larsen 1993)
2.61×목격된 심정지의 생존 가능성 (무목격 대비)
심뇌혈관 응급은 “분(分) 단위” 질환입니다. 그런데 절반이 목격자가 가장 적은 공간(집)에서 발생합니다. “쓰러졌는데 아무도 즉시 모르는” 신고 지연을 자동 감지로 없애는 것이 이 서비스의 핵심 가치입니다. (강원 등 의료취약지·독거 맥락 — screening/06·07 근거)

1. 전체 파이프라인

[아이폰 홈캠] 카메라 영상 │ ▼ [Layer 1] MediaPipe Pose Landmarker ── 사람 골격 33개 3D 마커 추출 (브라우저, 실시간) │ + COCO-SSD 객체 탐지 (사람·침대·소파 박스) ▼ [Layer 1.5] 휴리스틱 상태머신 ── "급격한 하강 + 몸통 수직→수평" 동작 감지 (트리거) │ NORMAL → SUSPECTED → DOWN → 무동작 3초 → 후보! ▼ ┌─────────────── 트리거 시에만 (병렬) ───────────────┐ ├─▶[Layer 2] 로컬 ST-GCN 낙상 모델 ── keypoint 시퀀스로 "낙하 동작" 확률 (로컬 파이썬) └─▶[Layer 3] Claude Vision ── 키프레임 보고 "정말 쓰러져 무반응인가" 판정 (VLM) │ ▼ [EmergencyEvent 발행] → (팀원 서버) → 환자 앱 / 주변인 앱 / 119 ← 큰 알람 + 취소 게이트

감지 파트(이 저장소)의 책임은 EmergencyEvent 발행까지. 이후 환자·주변인 폰으로의 라우팅과 119는 앱 서버(팀원 담당)가 처리합니다.

2. 3층 캐스케이드 — 각 레이어와 모델

LAYER 0 · 입력

카메라 (아이폰 연속성 카메라 / 웹캠)

아이폰을 Mac의 웹캠으로 연결. 브라우저 getUserMedia가 인식. localhost는 보안 컨텍스트라 권한 정상.
LAYER 1 · 포즈 + 트리거

MediaPipe Pose Landmarker (33 landmark) + 휴리스틱 상태머신

골격 33개(3D)를 실시간 추출하고, 동작으로 쓰러짐을 1차 감지합니다.
판정 규칙(동작 기반): ① 실제 누적 하강(net descent, 지터에 강건) AND ② 쓰러진 자세(몸통 수평 또는 박스가 옆으로 퍼짐) — 둘 다 만족해야 트리거. 가만히 있거나 얼굴만 카메라에 가까워도(박스가 넓어져도) 하강이 없으면 발동 안 함. 침대·소파 존은 억제, 바닥은 의심. (자세한 규칙·파라미터는 §4·§5)
LAYER 2 · 학습된 동작 인식

로컬 ST-GCN 낙상 모델 (MultiScaleTCNAttn)

낙상 완료(DOWN) 순간에 스냅샷한 최근 keypoint 시퀀스를 학습된 모델에 보내 "낙하 동작" 확률을 받습니다. (발행 후가 아니라 DOWN 순간에 잘라, 탄도적 급락이 창의 최신 프레임에 오게 함.) 휴리스틱이 놓치거나 헷갈리는 경우를 학습 모델이 보강.
이 노트북의 로컬 파이썬(FastAPI) 백엔드에서 PyTorch로 직접 추론 — 클라우드 없음. 입력이 우리 MediaPipe 33 keypoint와 정확히 일치(x, y, visibility)해 좌표 변환이 필요 없음.
LAYER 3 · 의미 확인 (VLM)

Claude Vision (multi-image + tool_use)

트리거 순간의 키프레임 몇 장을 Claude에 보내 “실제로 쓰러져 움직이지 않는 응급인가”를 구조화 JSON으로 판정. 요가·낮잠 등 오탐을 의미적으로 제거.
우리 스택이 이미 Claude API라 통합이 쉽고, "챗봇 이상의 에이전트 워크플로우"를 정면으로 충족. (키 없으면 자동 생략되고 앱은 정상 동작)

3. 레이어별 부품 이해 — 각 레이어가 무엇인가

파이프라인의 각 레이어에 쓰이는 부품이 무엇이고, 무엇을 입력받아 무엇을 내놓는지. 핵심은 L0·L1의 카메라/포즈/사물 모델에는 "쓰러짐" 개념이 없다는 것 — "쓰러짐"은 L1.5에서 우리가 규칙으로 정의하고(§4), L2·L3가 이를 학습모델·VLM으로 보강합니다.

LAYER 0 · 입력

카메라 (아이폰 연속성 카메라 / 웹캠)

  • 무엇: 홈캠 역할. 아이폰을 Mac의 웹캠으로 잡는 연속성 카메라(Continuity Camera), 또는 일반 웹캠.
  • 입력 → 출력: 실세계 영상 → 브라우저 getUserMedia가 <video> 스트림으로 제공. 매 프레임을 L1이 소비.
  • localhost는 보안 컨텍스트(secure context)라 카메라 권한이 정상 동작. 시연 땐 심사위원이 볼 홈캠 = 발표자 아이폰.
LAYER 1 · 포즈 + 사물

① MediaPipe Pose Landmarker (BlazePose)

  • 무엇: Google의 실시간 인체 포즈 추정 모델. 브라우저에서 WASM으로 구동.
  • 입력 → 출력: 카메라 프레임 한 장 → 신체 관절 33개의 (x, y, z, visibility). (코·눈·어깨·팔꿈치·손목·엉덩이·무릎·발목 등)
  • 우리 용도: 어깨·엉덩이 좌표로 몸통 각도(수직/수평)와 엉덩이 중심의 하강을 계산.
  • ⚠️ "쓰러짐"을 모름 — "여기 어깨/엉덩이가 있다"까지만. 낙하·자세 판단은 우리 규칙의 몫.

② COCO-SSD

  • 무엇: COCO 데이터셋으로 학습된 SSD 객체 탐지 모델(TensorFlow.js, 브라우저 구동).
    • COCO = Common Objects in Context. 일상 사물 80개 클래스(person, bed, couch, chair, tv…).
    • SSD = Single Shot MultiBox Detector. 이미지를 한 번 통과시켜 여러 사물의 박스+클래스를 동시에 예측하는 경량 탐지 구조(그래서 실시간).
  • 입력 → 출력: 프레임 → [클래스, 신뢰도, 박스(x,y,w,h)] 목록.
  • 우리 용도 2가지:
    • 사람 박스(person) — 몸 크기(bodyHeight)와 가로세로비(aspect) 계산의 기준. 포즈 관절만으로도 박스를 만들 수 있지만(fallback), 사람 박스가 더 안정적.
    • 구역 자동 감지 — bed·couch·chair를 찾아 "여긴 침대" 억제 구역으로. 침대에 눕는 건 응급이 아니므로 오탐 억제.
  • ⚠️ COCO 80클래스에 "fall"이 없음 → COCO-SSD도 쓰러짐을 직접 알려주지 못함.
LAYER 1.5 · 트리거

휴리스틱 상태머신 (우리가 작성)

  • 무엇: 학습 모델이 아니라 손으로 짠 규칙 엔진. L1의 관절·박스를 시간축으로 보고 "쓰러짐"을 판정.
  • 입력 → 출력: 프레임별 관절·박스 → 상태(NORMAL/SUSPECTED/DOWN…) + 쓰러짐 후보 이벤트.
  • 왜 필요: L0·L1이 "쓰러짐"을 안 주므로 누군가는 정의해야 함. 규칙이라 왜 울렸는지 설명 가능(블랙박스 아님)하고 임계값을 실시간 조절 가능.
  • → 신호·상태 전이·파라미터 상세는 §4·§5.
LAYER 2 · 학습된 동작 인식

② 로컬 ST-GCN 낙상 모델 (arisu04 · MultiScaleTCNAttn)

  • 무엇: 낙상 데이터로 실제 훈련된 딥러닝 분류기 — 우리 5개 레이어 중 "쓰러짐"을 학습한 유일한 모델.
  • ST-GCN이란: Spatial-Temporal Graph Convolutional Network. 관절을 그래프 노드로, 시간축을 따라 골격의 움직임 패턴을 학습하는 동작 인식 표준 구조. (한 프레임이 아니라 시퀀스를 봄)
  • MultiScaleTCNAttn이란: 이 구현체의 이름.
    • TCN = Temporal Convolutional Network, 시간 방향 1D 합성곱으로 동작의 시간 패턴을 포착.
    • MultiScale = 여러 시간창(64·128·256 프레임)을 동시에 봐 빠른 낙하~느린 붕괴를 모두 커버.
    • Attn(Attention/SE) = 중요한 관절·채널에 가중치. (그래프 einsum이 없는 순수 Conv2d라 로컬·ONNX 친화)
  • 입력 → 출력: MediaPipe 33관절 시퀀스 (x, y, visibility) → "낙하 동작 확률" + 지속(sustained) 여부.
  • 언제 자르나: DOWN 전이(낙상 완료) 순간에 최근 160프레임 버퍼를 스냅샷 → 급락이 창의 최신 프레임에 위치(무동작 대기로 희석되지 않음).
  • 어디서 도나: 이 노트북의 로컬 파이썬(FastAPI) 백엔드(localhost:8000/fall)에서 PyTorch로 직접 추론. 클라우드 없음.
  • 우리 용도: 트리거 순간 규칙 판정을 보강/교차검증(주 판정 아님). 지표·라이선스 불명이라 데모 보조로.
LAYER 3 · 의미 확인

③ Claude Vision (VLM)

  • 무엇: VLM(Vision-Language Model) — 이미지를 보고 언어로 추론하는 멀티모달 모델. 전용 낙상 분류기가 아니라 범용 시각-언어 이해.
  • 입력 → 출력: 트리거 순간의 키프레임 여러 장(multi-image) → "정말 쓰러져 무반응인가"를 구조화 JSON으로. (fallen / motionless / needsHelp / confidence / reason)
  • tool_use란: 모델이 자유 문장이 아니라 정해진 함수 스키마(JSON)로 답하게 강제하는 기능. 프로그램이 바로 파싱 가능 → 오류 없는 연동.
  • 어디서 도나: Next.js 서버 라우트 /api/confirm → Claude API. (ANTHROPIC_API_KEY 없으면 자동 "생략"되고 앱은 정상 동작)
  • 우리 용도: 요가·낮잠 등 규칙·학습모델이 놓친 오탐을 의미적으로 최종 제거. "챗봇 이상의 에이전트 워크플로우" 요건 충족.

4. 쓰러짐을 어떻게 정의했나 — 규칙 기반 상태머신

사전학습 모델이 "쓰러짐"을 안 주므로, 관절 좌표의 시간적 움직임으로 우리가 정의합니다. 3개 신호 + 상태머신(lib/collapse-state-machine.ts).

3개 신호 (매 프레임, 최근 시간창 기준으로 계산)

  • ① 낙하 (dropSignal) — 엉덩이 중심(좌·우 hip 중점)의 세로 위치가 시간창 안에서 실제로 얼마나 내려갔는가(net descent). 내려갔다 올라오면 상쇄되므로 순간 튐/지터에 강건. 몸 크기로 정규화해 거리와 무관. 기준 이상이면 낙하 인정.
  • ② 쓰러진 자세 (endedDown) — 지금 몸이 서 있지 않은가. 아래 둘 중 하나면 인정:
    • 몸통 수평: 어깨→엉덩이 선의 각도가 수평 기준 이상 (옆으로 넘어짐)
    • 박스 퍼짐: 사람 박스의 가로/세로 비가 기준 이상 (카메라 쪽/뒤로 넘어져 2D 각도가 안 나와도 박스가 옆으로 퍼지면 인정)
  • ③ 무동작 (immobility) — 쓰러진 뒤 일정 시간 엉덩이 이동이 미미한가. 움직이면 타이머 리셋.
핵심 안전장치: 트리거는 ①낙하 AND ②쓰러진자세를 모두 요구. 정적 자세나 근접 얼굴은 박스가 아무리 퍼져도 낙하(descent)가 없어 발동 안 함 → 오탐 방지. (예전엔 얼굴만 확대돼도 오탐났던 문제를 이 이중 게이트로 해결)
상태 전이: NORMAL ──(①낙하 + ②쓰러진자세)──▶ DOWN ──(③무동작 N초 유지)──▶ 후보 발행 ──▶ 앱 확인·119 │ │ └─(①낙하만, 자세 미충족)─▶ SUSPECTED ─┘ ▲ (곧 자세 충족 시 진행 / 아니면 복귀) │ ※ 침대·소파 구역이면 억제(눕는 건 정상) │ ※ 도중에 다시 일어서면(직립) ───────────────┘ NORMAL 복귀

앞 4개가 슬라이더인 이유: 주 감지가 학습 모델이 아니라 "규칙"이기 때문. 규칙의 임계값을 사람이 카메라 환경에 맞춰 직접 돌립니다. 순수 학습 모델이면 이런 노브가 없습니다. → 이것이 §6에서 규칙 주력 + 모델 보조를 택한 이유와 직결됩니다.

7. 왜 “캐스케이드”인가

경량·상시 트리거(포즈+휴리스틱)로 후보를 잡고, 무거운 확인(학습 모델 + VLM)은 트리거 순간에만 돌립니다.
  • 오탐↓: 단일 판정보다 확인 단계를 얹으면 앉기·눕기 오판이 크게 줄어듦 (SmartHome-Bench: VLM 체이닝 +11.62%p).
  • 골든타임: 상시 무거운 모델을 안 돌려 지연·비용·프라이버시를 아낌.
  • 안전판: 위양성 편향으로 “웬만하면 알림” → 앱의 취소 게이트가 오탐을 회수 (119 앞엔 항상 사람/무응답 확인).

8. 정직한 한계 (발표 방어)

  • 영상만으로 실신 vs 낮잠 구별·의학적 원인 진단은 불가 — 임상 gold-standard(의사)조차 주관적. 그래서 “진단”이 아니라 “위험한 붕괴 패턴 감지 → 신고 지원”으로 포지셔닝.
  • 낙상 감지 ≠ 심정지 감지. 심정지 특화(호흡/agonal breathing)는 로드맵.
  • 어두운 방·가림에서 포즈 정확도 저하 — 조명/배치 전제.
  • 모든 AI 응답에 디스클레이머: “본 정보는 참고용이며 의학적 진단이 아닙니다.” 응급 시 119 안내 최우선.

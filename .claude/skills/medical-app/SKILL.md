# 의료 AI 앱 공통 워크플로우

> 앱 컨셉과 무관하게 적용되는 공통 규칙(합성데이터·디스클레이머·PHI·응급). 아래 엔티티/필드는 대표 예시이며, **확정된 컨셉에 맞춰 조정**한다.

## 1. 합성(가짜) 의료 데이터 생성
실제 환자 데이터 절대 금지. 데모/테스트용 합성 데이터만 사용.
- 환자: 가명 + 랜덤 나이/성별/증상 (18개 HIPAA 식별자 포함 금지)
- 병원/약국: 지역명 + 진료과 + 운영시간 (공개 데이터 스타일)
- 증상 시나리오: 경증/중등도/응급 골고루

## 2. 데이터 모델 (FHIR R4 참고, MVP용 단순화)
- `Patient` — 가명, 나이대, 성별
- `Condition` — 증상/상태, 응급도(triage)
- `Encounter` — 방문/문진 세션
- `Organization` — 의료기관(병원/의원/약국/보건소), 진료과, 위치, 시간

## 3. 필수 디스클레이머 (모든 진단성 AI 응답 하단)
```
본 정보는 참고용이며 의학적 진단이 아닙니다.
정확한 진단은 의료진과 상담하세요.
응급 상황 시 즉시 119에 연락하세요.
```

## 4. PHI 비식별화 규칙
개인 식별자(이름·주민번호·연락처·주소·생년월일 등 18종)는 저장·로그·커밋 전 반드시 마스킹/제거.

## 5. 응급 처리
red-flag 증상 감지 시 다른 안내보다 먼저 119/응급실 안내를 노출.

> 관련: anthropics/healthcare 플러그인의 fhir-developer 스킬 + ICD-10/PubMed MCP 데이터 서버와 함께 쓰면 강력.

---
description: 합성(가짜) 의료 데이터 생성
---

`medical-app` 스킬 규칙에 따라 데모용 합성 데이터를 생성해줘. 실제 환자 데이터·개인정보 절대 금지.

**확정된 앱 컨셉에 필요한 엔티티를 생성**한다 (Supabase seed 또는 로컬 JSON). 컨셉이 아직 미정이면 아래 기본 예시로 시작:
1. 의료기관 목록(병원/의원/약국/보건소) — 지역명 + 진료과 + 위치 + 운영시간 (20~30건)
2. 증상/상태 시나리오 — 경증/중등도/응급 골고루 (15건)
3. 가명 환자 프로필 — 나이대/성별만, 식별자 없음 (10건)

FHIR 스타일 단순 스키마(Patient/Condition/Encounter/Organization)를 따르고, 18개 HIPAA 식별자가 포함되지 않도록 할 것.
